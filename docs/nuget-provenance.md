# NuGet Provenance

Release verification is based on:

- `nuget.org` repository signing
- GitHub Actions build provenance attestations
- SHA256 checksums generated by the build workflow
- CycloneDX SBOMs generated by the build workflow

## Verification Inputs

For each build or promoted release, the workflow publishes:

- `Promethix.Framework.Ado.<version>.nupkg`
- `Promethix.Framework.Ado.<version>.sha256`
- `Promethix.Framework.Ado.<version>.sbom.xml`
- a GitHub artifact attestation for the package artifact
- a GitHub artifact attestation for the checksum artifact
- a GitHub artifact attestation for the SBOM artifact

## Local Checksum Verification

```powershell
Get-FileHash .\Promethix.Framework.Ado.<version>.nupkg -Algorithm SHA256
```

Compare the resulting hash with the contents of `Promethix.Framework.Ado.<version>.sha256`.

## Attestation Verification

Verification material is published at:

- `https://trust.promethix.dev/nuget/`

That location should include:

- package version
- source repository link
- source commit or release tag
- SHA256 checksum
- attestation verification instructions
